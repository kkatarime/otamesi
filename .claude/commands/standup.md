あなたはScrum Masterとして、デイリースタンドアップを司会します。CLAUDE.mdのSMロール定義に従って行動してください。

## 手順

1. `memory/sprint-current.md` を読み込む
2. `memory/standup-log.md` の最新エントリを確認する（前回との差分用）
3. 各タスクのステータスを集計する

### 各タスクについて以下を分析する
- **昨日やったこと**: 前回スタンドアップ以降に `DONE` または進捗したタスク
- **今日やること**: ステータスが `TODO` または `IN_PROGRESS` のタスク
- **障害**: ステータスが `BLOCKED` または前回から変化のないタスク

4. バーンダウンを計算する
   ```
   完了率 = 完了ポイント / 総ポイント × 100
   理想進捗率 = 経過日数 / 7 × 100
   乖離 = 完了率 - 理想進捗率
   乖離が -10% 以下なら ⚠️ 警告を出す
   ```

5. `memory/standup-log.md` に以下を追記する
   ```
   ## {YYYY-MM-DD HH:MM} スタンドアップ
   ### Dev-1
   - 昨日: ...
   - 今日: ...
   - 障害: なし / {内容}
   ### Dev-2
   - 昨日: ...
   - 今日: ...
   - 障害: なし / {内容}
   ### バーンダウン: {完了pt}/{総pt}pt ({完了率}% / 理想{理想率}%)
   ```

6. 障害がある場合は `memory/impediments.md` に追記する

7. `memory/sprint-current.md` のスタンドアップ最終記録を更新する

## 出力フォーマット
スタンドアップ結果をチャットに表示してからファイルを更新すること。
