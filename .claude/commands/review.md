あなたはQA/Reviewer Agentとして、最新の実装をレビューします。CLAUDE.mdのQAロール定義に従って行動してください。

## 手順

1. `memory/sprint-current.md` を読み込み、ステータスが `REVIEW_PENDING` のタスクを特定する

2. 各 `REVIEW_PENDING` タスクについて以下を確認する：
   - `git log --oneline -10` で関連コミットを確認
   - `git diff main...HEAD -- src/` で変更差分を確認

3. 以下のレビュー基準で評価する：

   **コード品質**
   - 関数は50行以内か
   - 変数名・関数名が意図を明確に表しているか
   - 不要なコメントがないか

   **テスト**
   - テストファイルが存在するか
   - 主要なケース（正常・異常）がテストされているか

   **セキュリティ**
   - ユーザー入力のバリデーションがあるか
   - 機密情報がハードコードされていないか

   **DoD適合**
   - Definition of Done（レビュー通過・テスト通過）を満たしているか

4. `memory/sprint-current.md` のレビュー記録テーブルを更新する
   - `APPROVED`: 全基準を満たしている → タスクステータスを `DONE` に更新
   - `NEEDS_REVISION`: 修正が必要 → 具体的な修正指示をコメントに記録

5. レビュー結果をチャットに表示する

## 出力フォーマット
```
## QAレビュー結果

### T-XXX: {タスク名}
**判定**: APPROVED ✅ / NEEDS_REVISION ❌

**コード品質**: ✅/❌ {コメント}
**テスト**: ✅/❌ {コメント}
**セキュリティ**: ✅/❌ {コメント}

**修正指示**（NEEDS_REVISIONの場合）:
- {具体的な修正内容}
```
